Latis Intelligence Platform

App development roadmap · June 2026 — February 2027

Nine stages take Latis Intelligence Platform from current state through external brand-direct pilot and Phase 2 activation paths for the other use cases by February 2027. Latis is the unified platform; the Sales Deck Generator is one feature within it, used by Spectrum reps to produce client-facing sales decks. Brand-direct customers will access Latis without the Sales Deck Generator — every other intelligence function remains available to them. The split is governed by role-based access control (RBAC) in Stage 6.

Stages 1 through 4 all complete by end of July so Spectrum pilot and internal feedback begins August 1, 2026. Stages 2 and 3 run concurrently from July onward — Stage 3 (feature lock + DoD signoff + LLM grounding) runs longer than Stage 2 (AWS migration). Stages 7 and 8 run concurrently from November 2026 onward — Stage 8 (Brand-Direct Pilot) runs longer than Stage 7 (Phase 1 data-dependent features). Two continuous streams — user testing and QA — run alongside every stage. Five critical milestones gate the program.

The five Latis use cases Latis Intelligence Platform serves five go-to-market use cases. Each has a distinct buyer, consumption mechanism, and roadmap track. The program stages below describe how we build everything; this overview shows what we are delivering for each use case.

Access model · Sales Deck Generator and broader Latis

Latis Intelligence Platform is the unified product. The Sales Deck Generator is one feature within Latis, used by Spectrum reps to build client-facing decks. Stage 6 introduces RBAC so the platform can serve three distinct user types: Spectrum reps (full Latis access including the Sales Deck Generator), agency partners (Latis intelligence functions plus portfolio view; no Sales Deck Generator), and brand-direct customers (Latis intelligence functions scoped to their own brand; no Sales Deck Generator). Brand-direct users do not need to build decks for themselves — they need every other capability Latis provides.

Phase detail

Stage 1
Audit · ground truth
June 2026
Goal
Inventory the current app state, validate where the 1% sample is sufficient versus where full data is needed, and produce a ground-truth document of what is actually built, what works, and what is known broken before any further work starts.
Exit gate
Ground-truth audit document signed off by Charter and Blend.
Risk
Low. Internal sprint with discretionary timing within June.
Deliverables
  • Full app feature inventory — every page, component, and data dependency catalogued
  • 1% sample validation — features that work reliably on the sample versus those that need full data documented
  • Known issues backlog with priority and root cause
  • Charter Foundations team review of current numbers and labels
  • LLM grounding state documented per surface
Stage 2
Blend → Spectrum AWS migration
July 2026 (concurrent with Stage 3)
Goal
Migrate Latis from Blend's AWS environment to Spectrum's AWS environment to close the security gap before any external user accesses the platform.
Exit gate
Infosec sign-off and successful cutover verification.
Risk
Spectrum infosec timeline is outside the build team's control. Buffer the schedule accordingly.
Deliverables
  • Spectrum infosec review completed
  • IAM, VPC, secrets management, and network configuration provisioned in Spectrum AWS
  • Snowflake connectivity from Spectrum AWS verified
  • App cutover with no functional change to the user experience
Stage 3
Lock app features · DoD signoff · LLM grounding
June – July 2026 (concurrent with Stages 1–2, longer)
Goal
Lock the application features as they exist, complete the Definition of Done signoff for those features, and verify LLM grounding so the platform has a clean foundation before the full data lights up the Phase 1 enhancement work.
Exit gate
DoD signed off by Charter and Blend; LLM grounding verified against the golden test set; approved positioning language verified verbatim in every output.
Risk
Feature scope creep — stay disciplined about not adding features in this window. New requests get deferred to Stage 7 or Stage 9.
Deliverables
  • Feature freeze on the current app surface
  • DoD checklist closed for every page
  • LLM grounding verified against the golden test set
  • Approved positioning language verified verbatim in every output
  • UX fixes from the current feedback cycle shipped
  • 1% sample representativeness validated for audience discovery, category skew, co-visits, and trend direction
Stage 4
Full data footprint into app
July 2026 (completes before Aug 1)
Goal
Replace the 1% sample with full-footprint, de-sampled data for the Latis surfaces that require precision. This is the unlock for Stage 7's Phase 1 data-dependent features.
Exit gate
Numbers consistent across every surface and reconcile to the Foundations data source of truth.
Risk
Foundations data delivery schedule. Sequence the cutover around their availability.
Deliverables
  • Snowflake gold layer populated by Foundations team
  • De-sampled household counts, audience sizes, and reach figures across every client-facing surface
  • Source attribution labels with as-of dates on every data-bearing page
  • 1% sample retained where directional is sufficient, with extrapolation methodology disclosed inline
  • Cross-page reconciliation verified for every curated brand
Stage 5
Spectrum pilot & internal feedback
August 1 – September 2026
Goal
Put Latis in front of Spectrum reps in a controlled setting to validate utility before broader rollout. By this point the app is feature-locked (Stage 3) and full data is flowing (Stage 4) so reps are exercising the real product, not a placeholder.
Exit gate
Pilot users confirm Latis changes how they prepare for or run advertiser meetings.
Risk
Reps don't use it. Mitigation is a tight weekly feedback loop and willingness to shelve features that don't drive usage.
Deliverables
  • Auth layer live; public URL closed
  • Three named pilot users plus roughly 10 additional Spectrum reps onboarded
  • Weekly feedback cadence with documented intake
  • Server-side session save with shareable URL working
  • Feedback feeds into Stage 7 (Phase 1 features) priority backlog
Stage 6
Roles & role-based access control
October 2026
Goal
Build role-based access control so different user types see only the Latis features appropriate for their role. The Sales Deck Generator becomes a Spectrum-only feature here; all other Latis intelligence functions remain available to agencies and brand-direct customers under their respective access scopes.
Exit gate
Adversarial testing confirms no brand role can see another brand's data, and the Sales Deck Generator is unreachable to any non-Spectrum role.
Risk
One brand seeing another brand's data ends the program. Treat RBAC verification as a hard gate, not a soft milestone.
Deliverables
  • Three roles minimum: Spectrum rep (full Latis access including Sales Deck Generator), agency partner (Latis intelligence + portfolio view; no Sales Deck Generator), brand-direct user (Latis intelligence scoped to own brand only; no Sales Deck Generator)
  • IDP integration with Spectrum's identity provider
  • Sales Deck Generator gated behind the Spectrum role; not visible in nav for other roles
  • Audit logging of all role-gated access decisions
  • Adversarial testing of role boundaries by an independent reviewer

Stage 8
Brand-direct pilot
November 2026 – February 2027 (concurrent with Stage 7, longer)
Goal
Onboard external brand-direct customers to Latis Intelligence Platform under RBAC. They get every intelligence function — audience analysis, insights, your best play, summary — scoped to their own brand. The Sales Deck Generator is intentionally absent for this audience. The pilot starts as Phase 1 features are landing (Stage 7) and continues through February as the cohort expands.
Exit gate
Two brand-direct customers report measurable value, with success defined in advance by Charter sales leadership; cohort expanded to 5+ brand-direct customers.
Risk
Brand-direct usage patterns differ from internal Spectrum-rep usage. Plan capacity to triage feature requests that diverge from the internal pilot.
Deliverables
  • First two brand-direct customers onboarded with dedicated role-gated logins
  • Per-brand usage metrics tracked (logins, sessions, time-in-app, feature engagement)
  • Feedback loop with brand product owners
  • Expansion to five or more brand-direct customers once initial value is demonstrated
  • Production hardening for any feature requests that pass triage
Stage 9
Phase 2 feature development · UC1, UC2, UC3, UC5
December 2026 – February 2027
Goal
Build out the activation paths for the other use cases beyond UC4 — AI Model Training (UC1), Data Collaboration (UC2), Agentic Workflow Enhancement (UC3), and Managed Service Audience-Based Media via Spectrum Reach (UC5). Each use case gets a working pilot integration by end of February 2027.
Exit gate
At least one partner per use case is in pilot or integration testing: one DSP on embeddings (UC1), one RMN in Clean Room (UC2), one AI platform on MCP (UC3), one Spectrum Reach managed-service campaign live (UC5).
Risk
Four parallel build tracks. If staffing constrains to fewer tracks, sequence UC2 first (highest commercial value), then UC5, then UC1, then UC3.
Deliverables

FF-02 Audience Trajectory & Trend Discovery UC4 · Phase 2 · tab 3 Q1 2027 Full Spectrum + Brand Surface whether brand or category interest is growing, declining, or accelerating across the Charter footprint. Latis Context Pack · portable brief for AI tools Paste this into the system prompt of any LLM or AI agent that needs to understand Latis. Self-contained; assumes the reader knows nothing about Charter, Spectrum Reach, or Latis. Extended with the five use cases.

1
What Latis is

Latis is Charter Communications' behavioral intelligence platform. It turns the household-level browsing, device, and video signals already flowing across Charter's network into structured audience insights that advertisers can use. The product is built and operated by Charter Intelligence Ventures (CIV), an internal Charter business unit. Latis is sold to brand advertisers and to the agencies that buy media on their behalf.

Tagline: Intelligence. Alive.

2
The data underneath

Latis runs on three first-party Charter signal sources, all collected passively at the household level, with no surveys and no panels.

  • Domain browsing (Deepfield DNS) — what households visit on the open internet. Approximately 30–31.5 million households in the current footprint, scanning roughly 0.6 trillion DNS events per day across the universe.
  • Network and device telemetry (CUJO) — what connected devices are in the home and how they behave. Approximately 486M+ connected devices observed across the footprint.
  • Video viewership — linear and streaming viewing patterns. Approximately 91B+ video consumption events per year. Integration is in progress; full availability is a Phase 1A target.

All three are deterministic signals collected from the network itself. None of them comes from a survey, an opt-in panel, or a modeled lookalike. Charter's footprint covers 41 states.

3
What makes Latis different

Four claims define Latis's positioning. Every one of them is grounded in how the data is collected, not in branding choice.

  • Deterministic, not probabilistic. Every signal is a real observed behavior at a real household, not an estimate.
  • Household-level, not device-level. The unit of analysis is the home, so every device in the home contributes to one behavioral profile.
  • Always-on, not survey-based. Continuous network observation. No recall bias, no opt-in panel.
  • Pre-purchase intent, not post-transaction history. The data shows what households are doing before they decide.
4
Approved positioning language · verbatim, no paraphrase

These two statements are the official Latis positioning. They appear verbatim in any context where Latis describes its privacy stance, identity handling, or competitive position. No synonyms, no restructuring, no qualifiers.

"We transform signals, we do not expose behavior. We enable relevance without identity."
"Google = intent (search). Meta = affinity (like). Amazon = transaction (bought). Latis = behavior (what households are doing across the full home environment, before decisions)."
5
What Latis never says

Hard rules. Anything that violates them is suppressed before reaching a client. They apply to every output in every channel, including any LLM-generated content.

  • No personally identifiable information. No names, addresses, phone numbers, email addresses, household IDs, or device identifiers.
  • No claim about a specific named individual's behavior. Latis sees households and segments, not people.
  • No reference to a Latis client other than the one currently being discussed. No cross-engagement leak.
  • No invented numbers. Every figure is traceable to a query against the real data, a stated assumption with disclosed methodology, or a cited external source.
  • No "will" or "guaranteed" attached to campaign outcomes. Latis describes what is observed, not what will happen.
  • No demographic or behavioral claim about a client's audience that the client could already get from their own CRM or loyalty system, unless the index is at least 1.2x against the Charter average and the skew is disclosed.
  • No paraphrase of the positioning language above.
  • No mixing of Latis brand identity with Spectrum Reach or Charter consumer brands. Latis is distinct.
6
Tone and voice

Latis sounds like a senior analyst delivering a finding, not a salesperson making a pitch. Claims are stated declaratively when the data supports them, qualified honestly when support is partial, and omitted entirely when support is absent. Hedging is reserved for genuine uncertainty.

Latis is technically precise. It uses the right words for the right concepts (deterministic, behavioral, household-level, passive, signal). It avoids marketing fluff (game-changing, next-generation, AI-powered).

7
Vocabulary
  • Household (HH) — the unit of analysis. A single Charter-subscribed home with all its devices.
  • Signal — an observed behavior. A domain visit, a device fingerprint, a viewing event.
  • Vertical — a behavioral category. Retail & Shopping, Financial Services, Automotive, Sportsbook, Fitness, SVOD, CPG / Loyalty, and similar industry groupings.
  • Index — comparison to the Charter population average. 100 = same as average. 120 = 20% above average. Used everywhere Latis describes audience skew.
  • Co-visit — two domains visited by the same household. The foundation of cross-category insight.
  • Intender — a household showing in-market behavioral signals for a category (e.g., travel intender, auto intender).
  • Embeddings — high-dimensional vector representation of a household's behavioral state, generated by the Latis embedding model and pre-calculated daily.
  • Clean Room — Latis's Snowflake-based data collaboration environment. Partner first-party data is matched and enriched against Latis Intelligence Features within Latis's governed compute zone.
  • Blockgraph — the identity spine used by Latis to manage household identifiers. Spectrum Reach operates on Blockgraph natively.
  • MCP (Model Context Protocol) — the standard for connecting AI agents to external data sources. Latis exposes Intelligence Features via MCP for agentic workflow use cases.
  • Phase 0 / Phase 1A — current state vs enhanced state. Phase 0 covers approximately 15% of high-intent domain activity (current sampling). Phase 1A covers approximately 30% US household coverage (full footprint, post Snowflake provisioning).
8
What Latis is for

Latis is built to support a sales conversation between a Charter / Spectrum Reach rep and an advertiser (brand or agency). The product surfaces audience intelligence the advertiser cannot get elsewhere, organized into a persuasive narrative tailored to that advertiser. The goal of every Latis output is to give the rep a moment where the client says some version of "we think about you differently now."

Latis is not a media-buying tool, not a campaign-management platform, and not a measurement system. It is an intelligence layer that informs the conversation.

9
The five use cases · Latis go-to-market shape

Latis Intelligence Platform serves five distinct use cases, each with a different buyer and different consumption mechanism. The platform's modules map across these.

  • UC1 · AI Model Training. DSP and AI model teams consume Latis Embeddings as a training input layer. Delivered via API microservice (sub-50ms p99) or daily bulk S3 push. Buyer: DSP ML / data engineering teams, AI model teams. Example: Amazon DSP, Anthropic.
  • UC2 · Data Collaboration. Retail Media Networks and brand data partners combine their first-party data with Latis Intelligence Features inside the Clean Room. Identity translation to UID2, Mercury, Yahoo Connect, NextEra. Buyer: RMN product teams + agency partners. Example: Walmart Connect.
  • UC3 · Agentic Workflow Enhancement. AI agents query Latis Intelligence Features via MCP or API at decision time to enrich routing, personalization, recommendation. Buyer: AI product / platform engineering teams. Example: retail and device-based AI agents.
  • UC4 · Consumer Insights. Self-service web UI for brand analytics, insights, and strategy teams. The Latis app at latis.blend360.app is the surface for this use case. Buyer: brand-direct analytics teams. Example: CPG brands, direct-to-consumer advertisers.
  • UC5 · Managed Service Audience-Based Media. Latis builds audiences (Blockgraph, Snowflake clean room). Spectrum Reach runs campaigns on Charter linear + CTV inventory via Freewheel / Radar. Client does not interact with Latis directly. Buyer: brands and agencies purchasing through Spectrum Reach. Example: Spectrum Reach beta program.

The Sales Deck Generator (a feature within the Latis app for UC4) is used by Spectrum Reach reps to support selling UC5 to brands.
